## 关于 hymo 与 Hybrid Mount 项目的致歉与和解声明

我是 [hymo](https://github.com/Anatdx/hymo) 作者 Anatdx。针对此前与 [Hybrid Mount](https://github.com/Hybrid-Mount/meta-hybrid_mount)（meta-hybrid）之间的 DMCA 纠纷，声明如下。

## 致歉

一、**早期**：[hymo](https://github.com/Anatdx/hymo) 曾基于 [Hybrid Mount](https://github.com/Hybrid-Mount/meta-hybrid_mount) 相关代码进行开发，当时在署名与保留版权信息方面符合 GPLv3 的要求；后续在一次提交中移除了对 meta-hybrid 的引用（当时仅删除 README 中友链，未修改提交记录等），客观上削弱了原项目的可见度，也容易让人误解为刻意淡化来源。这些做法给 Hybrid Mount 作者带来了困扰与不快，我由衷歉意，并深刻反思自己在开源合规与尊重原创方面的不足。

二、**纠纷**：此事最终发展为 DMCA 投诉与反通知，占用了双方与平台的时间精力，也对开源社区氛围造成影响。无论对事实与法律如何认定，我都对走到这一步感到遗憾与愧疚，愿为化解矛盾、减少误会承担责任。若因我的疏忽或不当行为给 Hybrid Mount 作者带来任何不便或负面情绪，我在此再次致以歉意。

三、**协议**：鉴于重写后 hymo 模块的大部分内容在 GPLv3 意义上仍构成对 Hybrid Mount 的衍生作品，我已按照 Hybrid Mount 方所提要求，将整个项目整体升级为 GPLv3 协议并注明作者与贡献记录。同时，我保留对 hymofs 部分（内核外、本人原创实现）的协议制定权利。以上仅为本人对协议安排的理解与说明，不构成对任何法律问题的最终认定。

四、**澄清**：hymo 并未、也从未试图清洗或篡改提交记录，当时所做的仅为删除友链。随后因收到 DMCA 通知，为自保我暂时删除了 GitHub 仓库；但重新上传的重写内容仍被 DMCA 限制，因此我不得不提交反通知。仓库恢复后，我已从 fork 仓库中恢复全部提交记录。特此说明，以免误解。

## 和解意向

我无意与 [Hybrid Mount](https://github.com/Hybrid-Mount/meta-hybrid_mount) 对立，更希望以尊重、透明的方式处理历史与现状。事情发生时我接触开源社区仅两月不到，相关规矩不太懂，若此前言行有欠妥之处，恳请海涵。

- **尊重原创**：我尊重 Hybrid Mount 及其作者在架构设计、WebUI 与工程实践上的创作与贡献；愿在 hymo 的文档或界面中增加对 Hybrid Mount 的致谢与历史说明（在符合当前代码与法律立场的前提下）。
- **配合沟通**：若对方或社区对具体文件、历史提交仍有疑问，我愿意在合理范围内配合说明或沟通。
- **今后**：我会在后续开发与对外表述中，更加注意对上游与相关项目的署名与尊重，避免类似误会再次发生。

## 联系

若 [Hybrid Mount](https://github.com/Hybrid-Mount/meta-hybrid_mount) 作者愿意沟通或提出具体诉求（例如希望在何处以何种形式保留署名或说明），可通过 GitHub 或您方便的方式与我联系。我会认真对待，在合理、合法的前提下配合，寻求双方都能接受的解决方式。

*本声明为个人致歉与和解意向，不构成对任何法律事实或权利的承认或放弃，亦不构成对反通知等程序中已陈述立场的变更。*
